引用

强引用：

一个对象具有强引用，他就不会被垃圾回收器回收

内存不足时：JVM也不会回收它，而是抛出OutOfMemoryError

终端强引用和某个对象之间的关联，可将引用赋值为null。

eg：

Object strongRef = new Object( );

String str = "hello" (?str为啥也是引用)

str = null；
软引用：

内存空间足够就不会被垃圾回收器回收。

内存空间不足时软引用才会被垃圾回收器回收。

JDK提供SoftReference类

被垃圾回收时遵循LRU法则，优先回收最少被使用的对象。

SoftReference<String> softName = new SoftReference<>("作业好多")

注解

1. 作用在代码的常用注解

（1）@Override

检查该方法是否是重载方法。如果发现其父类，或者是引用的接口中并没有该方法时，会报编译错误。

（2）@Deprecated
标记过时方法。如果使用该方法，会报编译警告。
（3）@SuppressWarnings

指示编译器去忽略注解中声明的警告。

这几个注解都具有编译检查作用

2.作用在其他注解的常用注解(或者说 元注解):

（1）@Retention

标识该注解怎么保存，是只在代码中，还是编入class文件中，或者是在运行时可以通过反射访问。

（2）@Documented

标记这些注解是否包含在用户文档中。

（3）@Target

标记该注解应该是哪种 Java 成员。

（4）@Inherited

标记该注解继承于哪个注解类(默认 注解并没有继承于任何子类)
