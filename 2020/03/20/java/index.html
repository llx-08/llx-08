注解

简介

主要作用包括：

–生成文档。这是最常见的，也是java 最早提供的注解。

–跟踪代码依赖性，实现替代配置文件功能。比较常见的是spring 2.5 开始的基于注解配置。作用就是减少配置。

–在编译时进行格式检查。如@Override 放在方法前，如果你这个方法并不是覆盖了超类方法，则编译时就能检查出。

  1. 作用在代码的常用注解

（1）@Override

检查该方法是否是重载方法。如果发现其父类，或者是引用的接口中并没有该方法时，会报编译错误。

（2）@Deprecated

标记过时方法。如果使用该方法，会报编译警告。

（3）@SuppressWarnings

指示编译器去忽略注解中声明的警告。

这几个注解都具有编译检查作用

2.作用在其他注解的常用注解(或者说 元注解):

（1）@Retention

标识该注解怎么保存，是只在代码中，还是编入class文件中，或者是在运行时可以通过反射访问。

（2）@Documented

标记这些注解是否包含在用户文档中。

（3）@Target

标记该注解应该是哪种 Java 成员。

（4）@Inherited

标记该注解继承于哪个注解类(默认 注解并没有继承于任何子类)

Super:

关键字super用来引用当前对象的父类。

可以使用它访问父类中被覆盖的成员（比如参数名相同）。

基本格式：

(1) 访问父类被隐藏的成员变量，如：

super.variable;

​ (2) 调用父类中被重写的方法，如：

super.Method([paramlist]);

​ (3) 调用父类的构造函数，如：

super([paramlist]);

若类没有有参的构造方法，编译器会自动添加一个无参构造方法，在此方法中只有一条语句super()，若已有有参的构造方法，则不会自动添加

在任何构造方法中的第一条语句，若没有使用super调用父类的任何构造方法，编译器自动添加语句super()

这里就会有一个有意思的问题：

如果父类定义了有参数的构造方法（也就是说编译器不会为它产生默认的构造方法），此时当子类没有构造方法的时候，编译器为其生成了的构造方法内只有super（）【注意没有参数】，此时由于父类没有不含参数的构造方法，编译就会出错。（有一种奇怪的严谨性，但是好奇怪）

this：

抽象类不能被实例化，必须被继承

接口

一种协议，声称需要完成什么方法。

接口是抽象类的一种变体，用关键字interface来定义，形式为：

[修饰符] interface 接口名
{
   //变量和方法声明
}

接口中的所有方法都是抽象的（abstract可以不写），也没有方法体。

接口中的所有数据都是静态常量（static,final可以不写）。

//eg
interface MyTest{
    double i = 9.8;//等价于public static final double i=9.8
    void show();//public abstract void show
}

在类的声明中用implements来表示一个类实现自某个接口。

class 类名 implements 接口名 [extends 类名] {
    //类体
}

一个类可以实现多个接口,在implements子句中用逗号分开。如：class D implements A,B,C

实现某接口的子类必须重写该接口中所有方法。

//eg
interface MyTest{
    double i = 3,j = 4;
    void compute();
}
class Crect implements MyTest{
    public void compute(){
        System.out.println("面积为"+i*j);
    }
}
public class Test{
    public static void main(String args[]){
        Crect r = new Crect();
        r.compute();
    }
}

// 1. 不需要参数,返回值为 5  
() -> 5  

// 2. 接收一个参数(数字类型),返回其2倍的值  
x -> 2 * x

(int x, int y) -> x + y

public class Java8Tester {
   public static void main(String args[]){
      Java8Tester tester = new Java8Tester();

      // 类型声明
      MathOperation addition = (int a, int b) -> a + b;

      // 不用类型声明
      MathOperation subtraction = (a, b) -> a - b;

      // 大括号中的返回语句
      MathOperation multiplication = (int a, int b) -> { return a * b; };

      // 没有大括号及返回语句
      MathOperation division = (int a, int b) -> a / b;

      System.out.println("10 + 5 = " + tester.operate(10, 5, addition));
      System.out.println("10 - 5 = " + tester.operate(10, 5, subtraction));
      System.out.println("10 x 5 = " + tester.operate(10, 5, multiplication));
      System.out.println("10 / 5 = " + tester.operate(10, 5, division));

      // 不用括号
      GreetingService greetService1 = message ->
      System.out.println("Hello " + message);

      // 用括号
      GreetingService greetService2 = (message) ->
      System.out.println("Hello " + message);

      greetService1.sayMessage("Runoob");
      greetService2.sayMessage("Google");
   }

   interface MathOperation {
      int operation(int a, int b);
   }

   interface GreetingService {
      void sayMessage(String message);
   }

   private int operate(int a, int b, MathOperation mathOperation){
      return mathOperation.operation(a, b);
   }
}

正则表达式

正则表达式是一个字符串，用来描述字符串匹配的模式，可以进行字符串匹配、替换和分解。

一个正则表达式是由普通的字符（例如字符a到z）以及特殊字符（元字符）组成的文字模式，用以描述在查找文字主体时待匹配的一个或多个字符串。

正则表达式作为一个模板，将某个字符模式与所搜索的字符串进行匹配。

（很多地方都能用到哟）

    {

        Matcher m=p.matcher(str);
        boolean b = m.matches();
        System.out.println(b);
        System.out.println(m);
    }
}

Java提供了与每一种简单类型相应的类。本质上，这些类包装了（wrap）简单类型的数据。因此，它们通常被称作类型包装器或者包装器类。

数组复制

使用arraycopy( )方法可以将一个任意类型的数组快速地从一个地方复制到另一个地方。这比使用Java中编写的循环要快的多。

public class HelloWorld{
    public static void main(String args[])
    {
        byte a[]={65,66,67,68,69,70,71};
        byte b[] = {88,88,88,88,88,88,88,88,88,88};
        System.out.println("a="+new String(a));

        System.out.println("b="+new String(b));

        System.arraycopy(a,0,b,0,a.length);
        //从b数组的0号开始拷贝a数组从length个字符
        System.out.println("b="+new String(b));

        System.arraycopy(b,5,a,0,4);
        //从a数组的0号拷贝b的5号后面4个字符
        System.out.println("a="+new String(a));
    }
}

LinkedList类

LinkedList类实现List接口，提供了一个链式列表数据结构。

同时还支持栈、队列的操作：

–栈：push、pop方法

–队列：offer、poll方法

对象存储

–存储在链表中，其实现依赖于LinkedList的静态内部类Node。

–Node类表示链表中的一个元素，包括：

成员变量E item用以存储数据对象; E表示泛型的类型。

成员变量Node next用以指向链表的下一个元素

对比

ArrayList

–优点：读取速度快，get(index)是通过对数组下标访问实现的。

–缺点：添加很慢，因为添加数据在array中间时，需移动后面的数据；删除同理。另外，当添加时元素个数大于数组初始长度时，需扩容。

LinkedList

–优点：添加很快，通过更改Node中next的引用指向即可实现；删除同理。 没有长度限制

–缺点：读取速度慢，get(index)必须对链表进行遍历。

–另外，支持栈Stack和队列Queue的操作。

import java.util.ArrayList;
import java.util.LinkedList;
import java.util.List;

public class HelloWorld{
    public static void main(String[] args) {
        List[]  array = {new ArrayList(), new LinkedList()};
        for(List list : array){
            System.out.println(list.getClass().getName());
            long s = System.currentTimeMillis();
            for (int i = 0; i < 100000; i++)
                list.add(0, new Integer(i));
            System.out.println("add(ms): " + (System.currentTimeMillis() - s));

            s = System.currentTimeMillis();
            for (int i = 0; i < 100000; i++)
                list.get(i);
            System.out.println("get(ms):  " + (System.currentTimeMillis() - s));

            s = System.currentTimeMillis();
            for (int i = 0; i < 100000; i++)
                list.remove(0);
            System.out.println("remove(ms): "+(System.currentTimeMillis()-s));
        }
    }
}

HashSet类

HashSet类使用哈希表进行存储。

–实例化后生成一个数组，表示一片连续的存储空间。

–构造函数之一的HashSet(int initialCapacity, float loadFactor)可设置哈希表的初始容量、装载因子。

–添加、删除、查找元素的时间复杂度：最好时为O(1)，最差为O(n)，n为元素的个数；通常接近O(1)。

当add一个对象 obj 到HashSet时：

  1. 调用obj的hashcode()方法取得hashcode值，默认返回值是基于obj的内存地址；

  2. 基于hashcode求取该对象在以上数组中的存储位置，如除留余数法；

  3. 若该位置上已有对象，则调用obj的equals()方法判断是否相等；若相等则不添加，若不等则以链地址法解决冲突。

boolean contains(Object obj)查找过程同add。

当存储对象数大于容量和装载因子之积时，自动扩充容量。

数组的每个条目都是一个链表（有next）；另外的为开放寻址法

哈希表天生就是为了提高查找效率的、空间换取时间。装载因子越小，则冲突越小，效率越高

以数组为基础的容器都需要扩充，ArrayList也是。

String类中的hashCode计算方法还是比较简单的，就是以31为权，每一位为字符的ASCII值进行运算，用自然溢出来等效取模。

哈希计算公式可以计为$s[0]\times31^{n-1} + s[1]\times31^{n-2} + … + s[n-1]$

为什么取31为权，主要是因为31是一个奇质数，所以$31\times i=32\times i-i=(i<<5)-i$，这种位移与减法结合的计算相比一般的运算快很多。

TreeSet类

TreeSet类使用平衡二叉树进行存储

–具体使用的平衡二叉树为红黑树

–在平衡二叉树中，对象的存储必然是有序的

–添加、删除、查找数据的时间复杂度：O(log n)，n为元素的个数

对象只有实现了Comparable接口，才可以存放于TreeSet当中，如String类

–Comparable接口仅包含一方法int compareTo(Object o)，在当前对象大于、等于、小于o时分别返回正数、零、负数

–添加数据时若树中已有数据与其相等(通过compareTo方法判断)，则放弃添加

比较器

Comparable接口

–方法：int compareTo (Object o);在当前对象大于、等于、小于o时分别返回正数、零、负数

–内部比较器

–若一个类实现了Comparable接口，则“该类支持比较”。

Comparator接口

–方法： int compare(Object o1, Object o2);

–外部比较器

–当一个类不支持比较时（即，没有实现Comparable接口），可以建立一个外部比较器来实现比较的功能；或者，当想要用其他比较方法时，也可以使用外部比较器。

import java.util.Comparator;
import java.util.TreeSet;

public class HelloWorld{
    public static void main(String[] args) {
        Comparator<Point> c = new Comparator<Point>(){
            public int compare(Point o1, Point o2) {
                if(o1.x > o2.x)      return 1;
                else if (o1.x<o2.x) return -1;
                else          return 0;
            }
        };
        TreeSet<Point> ts = new TreeSet<Point>(c);
        ts.add(new Point(0, 0));    ts.add(new Point(1, 1));
        ts.add(new Point(1, 1));    ts.add(new Point(2, 2));
        System.out.println(ts.size());
        for(Point s : ts)
            System.out.println( s );
    }
}
class Point{
    int x=0, y=0;
    public Point(int x, int y){ this.x = x;  this.y = y; }
    public String toString() { return x+","+y; }
}

使用哈希表存储键值对

实例化后生成一个数组，表示一片连续的存储空间用以存储键值对。

当put一对key和value 到HashMap时：

  1. 调用key的hashcode()方法取得hashcode值

  2. 基于hashcode求取该键值对在以上数组中的存储位置

  3. 若该位置上已有键值对，则调用key的equals()方法判断与其key是否相等；若相等则覆盖，若不等则解决冲突

get(key)方法的过程基本同put方法的过程

HashMap与HashSet的区别：

–HashSet基于对象的hashcode存储 对象

–HashMap基于键的hashcode存储 键值对

–添加数据时若已包含有相同数据(通过equals方法判断)，则HashSet会放弃添加，而HashMap会进行覆盖

TreeMap

TreeMap类使用平衡二叉树存储键值对

–具体使用的平衡二叉树为红黑树

–在平衡二叉树中，键值对的存储必然是有序的

key只有实现了Comparable接口，键值对才可以存放于TreeMap当中；否则构造TreeMap时需传入Comparator对象

TreeMap与TreeSet的区别：

–TreeSet基于对象的比较方法存储对象

–TreeMap基于键的比较方法存储键值对

–添加数据时若已包含有相同数据(通过compare方法判断)，则TreeSet会放弃添加，而TreeMap会进行覆盖

Collections——通用方法

Object max(Collection c), Object min(Collection c) 返回该Collection中的最大值、最小值；需包含的对象实现Comparable接口，或传入Comparator对象
void sort(List list) 将该List进行排序；需包含的对象实现Comparable接口，或传入Comparator对象
void shuffle(List list) 洗牌：随机打乱一list的顺序
synchronizedList(List l)、synchronizedSet(Set s)、synchronizedMap(Map m) 转换成同步集合 （多线程安全）
